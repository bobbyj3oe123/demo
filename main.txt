- local git states refer to image 1

- git add moves files to the staging area

- remove git repository from your project with
  - rm -rf .git

- you can add files to staging area also by git add 'filename'

- can add commits without message inline but through the text editor with
  - git commit

- get commit history with
  - git log

- get commit history of one file use
  - git log -- "Filename"

- simple view of git log
  - git log --oneline --graph --decorate --all
    - alias create for top so now we use
    - git log history

- get commit history and show info of changes use
  - git show

- find out which files git is tracking use
  - git ls-files

- add a track file that is modified file use
  - git commit -am "Commit message"

- unstage a file use
  - git reset HEAD "filename"

- revert to the last good state of a file use
  - git checkout -- LICENSE.md

- list out all the global variables we set use
  - git config --global --list

- rename a file saved with git use
  - git mv "orginal file name" "new file name"
  - git mv "example.txt" "demo.txt"

- it you want to remove a file in the git respository
  - git rm "filename"
  - git rm demo.txt

- to add deletions not recognized by git since it red use
  - git add -u

- to add all modified files not recognized by git since it red use
  - git add -A
